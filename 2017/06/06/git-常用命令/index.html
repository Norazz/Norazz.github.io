git 常用命令

git的作用
  • 源代码管理工具(版本控制)
  • 源利于团队开发
  • 对于个人开发–对某个功能的修改操作
其他前端代码管理工具
  • svn
  • maven vss tfs
初始化

git init 在需要的文件夹下执行命令

向暂存区添加文件
  • git add 文件名 可以将新添加的和已修改的文件放到暂存区,☆注意☆：不包括删除
    • git add . 将仓库下面的没有被提交到暂存区的代码放到暂存区
    • git add * 将当前仓库中的所有的没有被提交到暂存区的代码放到暂存区
    • git add -u 或者 git add –update 将(修改和删除的文件)的操作告诉暂存区
    • git add -A 或者 git add –all 将所有的修改与删除和未跟踪的文件都告诉暂存区。
查看仓库文件中文件状态

git status

用户名和邮箱的配置
  • git config user.name xxx
  • git config user.email xxx
  • 可以通过–global选项实现在本机一次配置多次使用
向主仓库添加文件

git commit -m 备注信息

上传代码到服务器

git push 仓库地址 master

移除全局用户名

git config –global –unset user.name

将远程仓库的代码更新到本地

git pull 服务器仓库地址 master

将本地仓库代码更新到服务器中

git push 服务器仓库地址 master

保存服务器地址

git remote add 名字 服务器地址

删除服务器地址

git remote remove 名字

查看以保存的服务器地址

git remote

设置文件不让git管理

在.git同级目录添加一个文件，叫 .gitignore

  • 创建没有文件名的文件
    1 直接新建 “.gitignore.”
    2 bash命令：touch .gitignore
    • 打开.gitignore进行编辑，一行一行的添加不需要git管理的文件
      1 可以设置同类型的文件：.js .css
      2 也可以设置整个目录：
      a/ a目录下面的所有文件
      a/      .js a目录下面的所有js文件

打印日志

git log

* git log –oneline 每一次提交的(短)版本号，以及备注信息

版本操作

git reset –hard 提交的版本号(可以只写前几位)

git reglog
  • 可查看：每一次提交的信息；以及每一次版本回滚的操作；分支切换操作
git分支操作
  • git branch 查看所有分支即当前分支
  • git branch 分知名 在当前分支下创建新分支
    • 新分支会继承当前分支到目前为止的所有代码
  • git checkout 切换分支
    • 切换到指定的分支
  • git merge -m 分支名 备注信息
    • 合并指定分支到当前分支
  • git branch -d 分支名
    • 删除指定分支
