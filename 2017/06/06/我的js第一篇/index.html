js高级知识汇总

面向对象

面向过程

  • 面向过程是一种思维方式，就是将解决问题的关注点放在解决问题的每一个详细步骤上。

面向对象

  • 面向对象也是一种思维方式，就是讲解决问题的关注点放在解决问题所需要的一些列对象上。

面向对象的三大特性

  • 封装性
    将实现功能的具体步骤封装起来，只对外暴露接口，外界在使用的时候，只需要考虑接口的使用，而不需要关心功能的具体实现，这就是封装。
  • 继承性
    js中的继承，是指一个对象中没有的属性和方法，另一个对象中有，我们可以直接拿过来用，就是继承。
    • 继承的实现
      1. 混入式继承：用for-in遍历对象进行赋值；
      2. 原型继承：
        (1) 直接将构造函数的原型指向需要继承的对象；
        (2) 用for-in将需要继承的对象中的属性和方法赋值给构造函数的原型；
      3. 经典继承
        • var p = Object.create(human);
  • 多态性(js中没有多态)

构造函数

构造函数定义

构造函数也是函数，知识构造函数一般用来初始化对象。

构造函数注意事项

  • 如果手动写了return语句；
  • 将构造函数当普通函数调用。

原型

原型的概念

  • 在js语句中，每次创建构造函时，系统都会默认为这个函数创建一个相关联的空对象，这个空对象就是构造函数的原型。

原型的作用

  • 实现代码的共享。在原型中创建的成员，都可以和被与其相关联的构造函数创建的对象共享。

原型的访问方式

  • 构造函数.prototype
  • 对象.__proto__

原型的使用方式

  • 给构造函数的原型通过对象的动态特性，添加新的成员。
  • 直接给构造函数的prototype属性赋值一个新的对象。

原型的使用注意事项

  • 一般只将公用内容放在原型中；
  • 属性的访问原则：现在自身查找，若无则往原型中查找，直到null为止；
  • 若是给对象的属性赋值，则不会往原型中查找；
  • 若构造函数的原型发生改变，则改变前和改变后创建的对象的原型是不同的。

proto

  • 通过 对象.__proto__也可以访问到构造的原型。
  • 但是这个属性是非标准属性，有兼容性问题 不推荐使用！
  • 一般只在调试时使用

constructor

  • 系统默认创建的原型中会有constructor属性，指向原型的构造函数；
  • 自定义的构造函数中不会自动添加constructor属性，一般我们会手动添加constructor属性，以维护原型的三角关系的合理性；

原型链

概念

  • 在js中，对象都有原型，原型也是对象，因此原型也有原型，这样就形成了一个链式结构，即为原型链。

js中创建函数的几种方式

  • 函数声明
  • 函数表达式
  • 通过Function创建

Function构造函数

  • js中所有函数的构造函数都是Function,包括Function本身。

使用注意

  • 如果不给Function传递参数，则创建的函数为空函数。
  • 如果只传递一个参数，这个参数就是函数的函数体，参数的类型为字符串，同时参数的内容必须符合js的语法规范。
  • 如果传递多个参数，那么最后一个参数为函数的函数体，之前的参数均为创建出来的函数的形参。

函数的内置对象

  1. name
    • 表示函数的名称
  2. length
    • 表示函数的形参个数
  3. caller
    • 表示函数在哪个函数中被调用，这个caller指向的就是这个外部函数。如果函数在全局中调用，那么caller就是null.
  4. arguments
    • 定义：是一个伪数组，保存了传入函数的实参。
    • 内部属性
    1. callee：表示当前正在执行的这个函数；
    2. length: 表示传入的实参的个数。
    • 作用
    1. 可以实现匿名函数的递归：arguments.callee();
    • 与函数的属性arguments的区别
  5. caller属性和arguments属性需要在函数调用的时候，才能访问到内容；所以访问这两个属性一般都是在函数内部！

实例成员和静态成员

  1. 实例成员
    • 是指通过实例（对象）访问的成员。
  2. 静态成员
    • 是指通过构造函数访问的成员。

函数的递归

定义

  • 在函数内部直接或者间接的调用函数本身。

要素

  1. 自己调用自己；
  2. 有结束条件。

化归思想

  • 将一个问题由难化易，由繁化简，由复杂化简单的过程称为化归，它是转化和归结的简称。

递归的应用

  1. 求前n项和

  2. 求阶乘

  3. n的m次方

  4. 斐波那契数列

  5. 获取后代元素

作用域

定义： 作用域即为变量起作用的范围，在js中，只有函数有分割作用域的功能。

词法作用域(静态作用域)

  • js中的作用域都是词法作用域，即在js中，变量起作用的范围只与函数的声明位置有关，与函数的调用位置无关。

变量提升

  • 在js代码的预解析阶段，系统会将所有声明的变量和函数提升到其所在作用域的最前面。

变量提升的特殊情况

  1. 函数同名的情况
    • 同名的函数都会提升，但是后面声明的函数会将前面的函数给覆盖掉；
  2. 变量与函数同名的情况
    • 只提升函数，忽略变量的声明；
  3. 预解析分作用域
    • 变量只会提升到其所在作用域的最前面；
  4. 预解析是分段的，“段”指的是“script”；
  5. 条件式函数声明
    • 在条件语句中声明函数，可以当做函数表达式来处理，只提升函数名，函数体留在原地。
  6. 变量或者函数与所在函数的形参同名的情况
    • 函数中形参的声明和赋值过程优先于变量提升，并且不参与变量提升。

作用域链

  • 定义
    • 在js中，函数可以创建作用域，而函数中又可以嵌套函数，这样就形成了作用域嵌套作用域的结构，这种链式结构就是作用域链。
  • 变量搜索原则
    • 现在自身作用域中查找，若找到则直接使用当前作用域内的变量，若找不到则继续向外层作用域查找，若找到就使用，若找不到就继续想外层查找，直到找到全局为止，若全局中依然没有，则报错。

闭包

定义

  • 在js中，闭包是一种特殊的函数，包裹在其他函数内部，拥有独立变量。

  • 保护变量，给变量赋值提供专有的渠道，在这个专有的渠道中，可以设置一些校验的逻辑，可以保证赋值的合法性！
  • 给函数提供一个私有的变量。

函数调用模式

    1. 函数调用模式
    • 函数名(),相当于window调用，this指向window；
    1. 方法调用模式
    • 对象.方法名(),this指向调用方法的对象；
    • 对象[方法名](),this指向调用方法的对象；
    1. 构造函数调用模式
      new 构造函数名(),this指向new创建的对象；

上下文调用模式

  1. call
    • 使用方式：函数名.call(要让this指向的对象, arg1…argN);
    • 执行过程
    1. 调用该函数
    2. 把函数中的this改变为call方法的第一个参数
    3. 会把call方法传入的第二个及以后所有的参数依次传递给函数作为实参！
  2. apply
    • 使用方式： 函数名.apply(要让this指向的对象, 数组或者伪数组);
    • 执行过程
    1. 调用该函数
    2. 把函数中的this改变为apply方法的第一个参数
    3. 会把apply方法传入的第二个参数的数组或者伪数组中的内容，拆解开依次传递给函数作为实参
  3. 两者的区别：传入参数的方式不同，apply传入数组或者伪数组
  4. 注意事项
    • call/apply方法如果第一个参数不传，则函数中的this会指向window对象！
    • 如果call/apply方法第一个参数传入的值类型的数据，则会自动将其转换成对应的引用类型的数据，再将this指向这个引用类型的数据！
    • 如果call/apply方法第一个参数传入的是null或者undefined那么this将会指向window对象。
  5. 使用举例： Object.prototype.toString.call();
  6. 上下文调用模式的应用
    • 将伪数组转化为数组
    • 借用构造函数继承

工厂模式与寄生模式的区别

Object.defineProperty(obj,propName,descriptor);

  • obj: 要给哪个对象添加属性
  • propName: 要添加的属性名称

  • descriptor: 这个属性的描述信息（特征） 是一个对象

    • enumerable: 设置属性能不能被遍历，默认为false
    • writable: 设置属性能不能被赋值，默认为false
    • configurable: 设置属性能不能被删除，默认为false

    • value： 设置属性的值

    • get方法：属性在被获取值的时候，会自动调用这个get方法

    • set方法：属性在被设置值的时候，会自动调用这个set方法

    • set方法和get方法不能同时和value还有writable一起出现

严格模式

“use strict”

  • 在严格模式下声明变量必须使用var
  • 严格模式下不允许函数的形参重名
  • ES6之前的严格模式下，属性不可以重名
  • 在严格模式下不允许使用8进制数字
  • eval在严格模式下有自己的作用域
